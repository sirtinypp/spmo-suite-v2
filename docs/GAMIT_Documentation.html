3. Feature Modules

3.1 Dashboard  ·  3.2 Asset Inventory  ·  3.3 Asset Detail

  • Dashboard: Total assets, valuation, class/status breakdowns, recent activity
  • Inventory: Paginated, searchable list with filters; clickable rows; Add Asset (staff); Import/Export
  • Detail: 4-tab interface (Property, Finance, Lifecycle, COA) with role-based modal editing, audit trail, cross-office notifications, service history, print buttons

3.4 Transactions

A. Inspection: Select asset → attach docs → Pending → Approved/Rejected
B. Batch Procurement: Line items + PO/PR docs → 9-state PAR Workflow (§4)
C. Transfer: Select asset → new officer + ITR → Pending → Approved/Rejected

3.5–3.7 History, Service Logs, Reports

  • Unified transaction history with approve/reject actions and pending badge
  • Per-asset service & maintenance logs (add/delete/print)
  • PDF reports: IAR, PAR (per-asset with signatures), Service Record

Workflow Rules

  • Each transition requires a specific role (Django groups)
  • Approver must have digital signature uploaded
  • Document validation at gates (PO/PR at submit, DR/SI at delivery)
  • Every transition → ApprovalLog (signature, timestamp, IP)

PAR PDF Generation

  • Draft: Inspector + Supervisor sigs, “DRAFT” watermark
  • Final: All signatures, no watermark, SHA-256 hashed
  • A4 template at static/inventory/pdf/par_template.pdf
